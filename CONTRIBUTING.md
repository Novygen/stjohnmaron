# Contributing to St John Maron Parish Website

We appreciate your help to make this project better. Below are guidelines to ensure a smooth contribution process.

## Table of Contents

1. [Issues](#issues)
2. [Branches](#branches)
3. [Pull Requests](#pull-requests)
4. [Coding Standards](#coding-standards)
5. [Local Setup](#local-setup)
6. [Contact](#contact)

## Issues

- Search for existing issues before creating a new one.
- Provide clear details, steps to reproduce, and environment info.

## Branches

- **main**: Always stable and production-ready.
- **feature/** or **bugfix/**: Use descriptive branch names for new features or bug fixes.
- Submit pull requests against `main` unless instructed otherwise.

## Pull Requests

- Fork the repo, create a feature branch, and commit your changes.
- Ensure your code passes lint checks and unit tests.
- Provide a clear description of your changes in the PR.
- Stay open to feedback and make adjustments as requested by reviewers.

## Coding Standards

- **TypeScript**: Follow recommended best practices.
- **Tailwind CSS**: Keep styling consistent and mobile-first.
- **Mongoose**: Keep DB connections in `utils/db.ts`.
- **Comments**: Document your code, especially any complex logic.

## Local Setup

1. Clone the repo.
2. Install dependencies:
   ```bash
   npm install
   ```
